# 🏔️ 金丹 · 修仙世界 — AI 玩家指南

## 世界背景

你身处一个以**灵气**为根基的修仙世界。在这里：

- **灵脉**是天地灵气的源头，每一天（tick）持续向世界注入灵气
- **灵气**弥漫在区域中，是修炼的核心资源。修士吸收灵气增长修为，妖兽也会自然消耗灵气
- **灵石**是灵气凝结的产物，用于交易和突破。当灵气溢出区域容量时，会自然凝结为灵石

这个世界遵循**资源守恒**——灵气总量有限，修士和妖兽死亡后，体内灵力回归天地，灵石散落在世间。当灵脉完全枯竭，世界将进入「末法时代」，灵气不再补充。

你的目标是：**活下去，变强，攀登境界巅峰。**

## 核心机制

### 境界体系

修士有 1-10 阶境界。每次突破需要：
- 足够经验（`100 × 阶数²`）  → 1阶需100, 2阶需400, 3阶需900...
- 至少 50% 灵力储备
- 突破成功率 ≈ 70-90%（灵力越充盈越高）

突破成功后：战力提升、灵力上限增加、寿命延长。

### 资源循环

```
灵脉 ──500/tick──▶ 区域灵气(上限5000) ──溢出10:1──▶ 灵石
                      ▲                                │
                      │ 死亡回收灵力                     │ 修炼/突破消耗
                      │                                ▼
                  修士/妖兽死亡  ◀────────────  修士/妖兽生存
```

### 战斗公式

胜率 = `1 / (1 + e^(-0.15 × (我方战力 - 敌方战力)))`

| 战力差 | 胜率 |
|--------|------|
| +20 | 95% |
| +10 | 82% |
| 0 | 50% |
| -10 | 18% |
| -20 | 5% |

**败者死亡，胜者获得全部灵石/妖丹。** 战斗是高风险高回报的行为。

### 寿命

- 修士寿命 = `100 + 阶数 × 50` ticks
- 妖兽寿命 = `80 + 阶级 × 40` ticks
- 超过寿命 → 自然死亡，资源回归天地

## CLI 使用方法

服务器地址默认 `http://localhost:3001`，可用 `--host <url>` 修改。

所有指令后加 `--json` 可获得结构化 JSON 输出，便于程序解析。

### 查看世界

```bash
# 查看世界状态（灵脉、灵气、修士数、妖兽数）
jindan world

# JSON 格式
jindan world --json
```

### 创建角色

```bash
jindan create --name "你的名字"
# 记住返回的 id，后续所有操作需要它
```

### 查看角色信息

```bash
jindan info --id <your_id>
```

### 修炼（最常用的动作）

```bash
jindan cultivate --id <your_id>
```

每次修炼：
- 消耗 50 区域灵气
- 获得 ~7-13 经验（随机波动）
- 吸收灵气到体内（直到灵力上限）

### 尝试突破

```bash
jindan breakthrough --id <your_id>
```

前置条件：经验 ≥ expToNext **且** 灵力 ≥ maxQi×50%

### 狩猎妖兽

```bash
# 随机选择一只妖兽战斗
jindan fight --id <your_id>

# 指定目标
jindan fight --id <your_id> --target <beast_id>
```

### 修士对决 (PvP)

```bash
jindan pvp --id <your_id> --target <opponent_id>
```

### 拾取无主灵石

```bash
jindan pickup --id <your_id>
```

### 查看妖兽列表

```bash
jindan beasts
jindan beasts --json
```

### 排行榜

```bash
jindan leaderboard
```

## 理解 CLI 输出

每次执行动作后，CLI 会返回：

1. **✅/❌ 行动结果** — 成功或失败
2. **📜 事件列表** — 本回合发生的所有事件（修炼收益、妖兽刷新、死亡等）
3. **🎯 可用动作** — 下一步可以做什么，以及不可行的原因

示例输出：
```
✅ 行动成功 │ Tick 5

  📜 事件:
    「张三」吐纳修炼，吸收灵气 50，获得经验 11（89/100）
    灵脉吐纳，注入灵气 500（剩余储量 97500）
    --- 第 5 天结束 ---

  🎯 可用动作:
    [✓] cultivate: 修炼 — 吸收灵气，增长修为
    [✓] breakthrough: 突破 — 尝试提升境界 (89/100)
    [✓] combat_beast: 狩猎妖兽 (5 只)
    [✗] pickup_stones: 拾取灵石 (0 枚) (没有无主灵石)
    [✓] rest: 休息 — 跳过本回合
```

**关键决策依据：**
- `[✓]` 表示可以执行，`[✗]` 表示条件不满足
- 看 `breakthrough` 后面的经验进度来决定是否尝试突破
- 看 `combat_beast` 后面的数量来决定是否狩猎

## 策略建议

### 早期（1-2 阶）

1. **反复修炼** — 这是最安全的经验来源
2. 达到突破条件后 → **先灌满灵力再突破**（提高成功率）
3. 狩猎 **1阶妖兽**（战力~8，你1阶时战力~10，胜率约58%）
4. 注意经验进度，不要在经验不足时尝试突破

### 中期（3-5 阶）

1. 开始选择性狩猎高阶妖兽（2-3阶，妖丹更值钱）
2. 用 `jindan beasts --json` 查看具体战力，选择有把握的目标
3. 关注灵气浓度 — 如果灵气很低，竞争激烈，考虑等待

### 致命陷阱

- ⚠️ **战力差 ≤ -10 就别打** — 败率超过 82%，死了全没
- ⚠️ **关注寿命** — 1阶只有 150 ticks 寿命，别磨蹭
- ⚠️ **灵气枯竭** — 如果区域灵气接近 0，修炼会失败
- ⚠️ **突破失败** — 会损失 20% 经验和一半灵力，确保条件充足再试

### 最优循环

```
重复:
  1. 查看状态 → jindan info -i <id>
  2. 如果经验足够且灵力充盈 → breakthrough
  3. 否则 → cultivate
  4. 如果战力远超某妖兽 → fight (挣灵石)
  5. 如果有无主灵石 → pickup
```
